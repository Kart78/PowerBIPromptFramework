The PBIP + AI Workflow

Save your report as a .pbip project. Every visual becomes a JSON file. Every measure becomes readable text. AI can read that — and write it. This framework gives you the exact prompts to do it right.

01
Save as .pbip
File → Save as → Power BI Project. Unlocks JSON structure for all visuals and measures.
02
Load Claude Skill
Point Claude to the PBIR skill library so it knows native visual JSON schemas.
03
Give Context
Share your data model, tables, columns, and report goal in natural language.
04
Build & Place
Claude writes measures, builds pages, and places visuals directly as JSON files.
05
Reload Desktop
Reload the .pbip in Power BI Desktop. Report appears fully built. Iterate as needed.

PHASE 1
Context Prompts

Before Claude can build anything, it needs to understand your data model. These prompts establish the foundation — tables, columns, relationships, grain, and business rules.

P-01 Data Model Introduction
CONTEXT ▼
Use this first. Establishes the full schema before any build requests. Paste in your actual table/column names.
# DATA MODEL CONTEXT You are a Power BI developer working with the PBIR format (.pbip project). ## Project Location My .pbip project is located at: [path/to/project.pbip] ## Data Model The model contains the following tables: [TABLE NAME] — [description, e.g. "Fact table with one row per sales order line"] Key columns: [Col1, Col2, Col3] Grain: [e.g. one row per transaction per day] [DIM TABLE NAME] — [description] Key columns: [Col1, Col2, Col3] ## Relationships - [FactTable[DateKey]] → [DimDate[DateKey]] (many-to-one) - [FactTable[ProductID]] → [DimProduct[ProductID]] (many-to-one) ## Business Rules - [e.g. Revenue = SUM(Sales[Amount]) excluding returns] - [e.g. Budget is stored in a separate BudgetFact table] Confirm you understand the model before I ask you to build anything.
[path/to/project.pbip] [TABLE NAME] [description] [Key columns] [Relationships] [Business Rules]
P-02 Load PBIR Skill Library
CONTEXT ▼
Points Claude to the PBIR JSON skill so it can write valid visual definitions. Run this once per session.
Before we begin building, please read the PBIR skill reference at: [path/to/pbir-skill/SKILL.md] This teaches you the JSON schema for native Power BI visuals including: - Visual type identifiers - Config structure (x, y, width, height) - Data role bindings (category, values, legend) - Filter and slicer JSON patterns - Measure definition format Once loaded, confirm which visual types you have reference JSON for.
[path/to/pbir-skill/SKILL.md]
P-03 Report Goal & Audience
CONTEXT ▼
Sets the strategic intent. Claude needs to know WHY the report exists to make good layout and visual choices.
## Report Purpose This report is for: [audience — e.g. "C-suite monthly business review"] Primary question it must answer: [e.g. "Are we on track vs. budget this month?"] Report style: [e.g. IBCS-compliant / executive / operational / exploratory] Key metrics the audience cares about: 1. [Metric 1 — e.g. Revenue vs Budget] 2. [Metric 2 — e.g. Gross Margin %] 3. [Metric 3 — e.g. Units Sold YTD] Slicing dimensions: [e.g. Region, Product Category, Time Period] The report should have [N] pages: - Page 1: [Overview / Summary] - Page 2: [Drill-down / Detail]
[audience] [primary question] [report style] [metrics] [pages]
PHASE 2
Build Prompts

These prompts instruct Claude to write measures, create report pages, and place visuals as PBIR JSON files directly into your project folder.

P-04 Write DAX Measures
BUILD ▼
Ask Claude to write all required measures before building visuals. Always do this first in the build phase.
Based on the data model above, write the following measures and save them to: [ProjectFolder/SemanticModel/definition/tables/MeasureTable/] Measures needed: 1. [Measure Name] Logic: [e.g. Total Revenue = SUM of Sales[Amount] where Status = "Complete"] Format: [e.g. Currency, 0 decimals] 2. [Measure Name] Logic: [e.g. Revenue vs Budget % = DIVIDE(Revenue - Budget, Budget)] Format: [e.g. Percentage, 1 decimal] 3. [Measure Name — time intelligence] Logic: [e.g. Revenue YTD using TOTALYTD against DimDate[Date]] Write clean, commented DAX. Use variables (VAR) for readability. After writing, list all measure names so I can reference them in visual bindings.
P-05 Build a Full Report Page
BUILD ▼
The core build prompt. Creates a complete page with multiple visuals as PBIR JSON. Be explicit about layout positions.
Create a new report page named "[Page Name]" in the PBIR project at: [ProjectFolder/Reports/ReportName/definition/pages/] Canvas size: [1280 x 720] (standard 16:9) Place the following visuals: --- VISUAL 1: KPI Card --- Type: KPI card (native) Position: x=[20], y=[20], width=[200], height=[100] Measure: [Revenue] Comparison measure: [Budget Revenue] Label: "[Revenue]" --- VISUAL 2: Line Chart (Trend) --- Type: Line chart Position: x=[240], y=[20], width=[600], height=[200] X-axis: [DimDate[MonthYear]] Y-axis values: [Revenue, Budget Revenue] Legend: Show Title: "[Monthly Revenue vs Budget]" --- VISUAL 3: Slicer --- Type: Slicer (dropdown) Position: x=[860], y=[20], width=[200], height=[100] Field: [DimRegion[RegionName]] Write the complete PBIR JSON for each visual and the page definition file.
P-06 IBCS Variance Visual
BUILD ▼
Specifically for IBCS-style bar/waterfall variance charts — the most common advanced visual in executive reporting.
Add an IBCS-style variance chart to page "[Page Name]": Visual type: [Clustered bar / Waterfall] Position: x=[20], y=[140], width=[580], height=[260] Data bindings: - Category axis: [DimProduct[Category]] - Actual value: [Revenue] measure - Reference/Plan value: [Budget Revenue] measure - Variance measure: [Revenue vs Budget Variance] IBCS formatting rules to apply: - Actuals bar color: [#404040 (dark gray)] - Plan bar: outlined only, no fill - Positive variance: [#3ecf8e (green)] - Negative variance: [#e84848 (red)] - Data labels: show absolute variance values - No chart border, no gridlines - Font: consistent with report theme Apply these formatting overrides in the visual JSON config block.
P-07 Add Conditional Formatting
BUILD ▼
Applies rules-based or measure-driven conditional formatting to table or matrix visuals via JSON.
Apply conditional formatting to the [table/matrix] visual on page "[Page Name]": Column to format: [Revenue vs Budget %] Format type: [Background color / Font color / Icon] Rules: - If value < [-0.05]: color = [#e84848] (red — significantly below) - If value between [-0.05] and [0]: color = [#f5c518] (amber — slightly below) - If value >= [0]: color = [#3ecf8e] (green — on or above target) Apply this as a rules-based conditional format in the visual JSON. If icon set is used, use: [up/flat/down arrows].
PHASE 3
Refine Prompts

Once the structure is built, use these prompts to fine-tune layout, formatting, labels, and interactions without rebuilding from scratch.

P-08 Resize & Reposition Visual
REFINE ▼
Precise pixel-level repositioning. Much faster than drag-and-drop when you know the desired coordinates.
In the PBIR JSON for page "[Page Name]", update the position of the "[visual title or type]" visual: Current position: x=[old_x], y=[old_y], w=[old_w], h=[old_h] New position: x=[new_x], y=[new_y], w=[new_w], h=[new_h] Update only the position/size properties in the config JSON. Do not change any data bindings or formatting settings.
P-09 Apply Report Theme
REFINE ▼
Applies a consistent color palette and typography across all visuals via the theme JSON file.
Update the report theme file at: [ProjectFolder/Reports/ReportName/StaticResources/RegisteredResources/] Apply the following theme settings: Brand colors (in order): 1. [#your-primary] — primary / actuals 2. [#your-secondary] — secondary / plan 3. [#your-accent] — positive variance 4. [#e84848] — negative variance / alert 5. [#6b7090] — neutral / muted Background: [#ffffff / #f4f5f7] Font family: [Segoe UI / DIN / Calibri] Default font size: [10] Write the complete theme JSON and confirm file path.
P-10 Update Data Labels & Titles
REFINE ▼
Fine-tune how numbers and titles appear on visuals without touching the data model.
For the "[visual name]" on page "[Page Name]", update: Title: - Text: "[New Title Text]" - Font size: [12] - Bold: [true/false] - Alignment: [left/center] Data labels: - Show: [true] - Format: [#,0 / 0.0% / $#,0K] - Font size: [9] - Position: [inside end / outside end / callout] Axis labels: - X-axis title: [hidden/shown, text] - Y-axis: [hidden — use data labels instead] Update only the formatting config JSON for this visual.
P-11 Add Cross-Filter Interaction
REFINE ▼
Configure which visuals filter which others — critical for executive dashboards with slicers.
Configure visual interactions on page "[Page Name]": When the user clicks "[Slicer / Chart name]": - "[Visual A]" → [filter / highlight / no interaction] - "[Visual B]" → [filter / highlight / no interaction] - "[Visual C]" → [no interaction] (keep static) Apply these interaction settings in the page definition JSON. Note: "filter" removes non-matching rows; "highlight" dims them.
PHASE 4
Debug Prompts

When something doesn't load, looks wrong, or throws an error in Power BI Desktop — use these prompts to diagnose and fix the PBIR JSON.

P-12 Visual Won't Load / Blank
DEBUG ▼
The most common issue — visual appears blank or "can't display" after reload. Usually a data role binding or schema mismatch.
The "[visual type]" on page "[Page Name]" is not rendering after reload. It shows: [blank / "Can't display" error / loading spinner that never resolves] Here is the current visual JSON: [paste visual JSON here] Please diagnose: 1. Are the data role names correct for this visual type? 2. Are all referenced measures/columns spelled exactly as they appear in the model? 3. Is the visual type identifier correct? 4. Is the config structure valid for PBIR format? Provide the corrected JSON with comments explaining each fix.
P-13 Measure Returns Error
DEBUG ▼
When a measure shows [Error] or unexpected blank in Power BI. Paste the DAX and describe the expected output.
This DAX measure is returning [an error / blank / wrong value]: [Measure Name] = [paste DAX here] Expected result: [e.g. "Should return current month revenue divided by budget, as a decimal"] Actual result: [e.g. "Returns blank when filtering by Region"] Context: the measure is used in a [bar chart / table / KPI card] filtered by [Region slicer] Known model facts: - [e.g. Budget table has no Region column — joined via DimProduct] Diagnose the issue and rewrite the corrected DAX.
P-14 JSON Validation Check
DEBUG ▼
Power BI Desktop fails to load the .pbip project entirely — usually a malformed JSON file in the project structure.
Power BI Desktop won't reload my .pbip project after the last changes. The error says: [paste error message if any] The last files I modified were: - [file path 1] - [file path 2] Here are their current contents: [File 1 name]: [paste JSON] [File 2 name]: [paste JSON] Validate the JSON structure for PBIR compliance. Check for: trailing commas, missing required fields, incorrect nesting, invalid property names. Return corrected files with changes highlighted.

BEST PRACTICES
What Makes AI PBI Dev Work

Hard-won lessons from using Claude + PBIR in production. The quality of AI output is directly proportional to the quality of your inputs.

📐
Always specify coordinates
Give exact x, y, width, height values in every visual build prompt. Claude can't "see" the canvas — without coordinates, overlap is guaranteed.
🧱
Build measures first
Never ask Claude to build a visual and write the DAX in the same prompt. Write measures first, confirm they exist, then bind them to visuals.
📖
Load the skill every session
Claude doesn't retain the PBIR skill between conversations. Always run P-02 at the start of each new session before any build prompts.
🔤
Spell field names exactly
PBIR JSON is case-sensitive and exact-match. Copy-paste column and measure names directly from Power BI. One wrong character = blank visual.
🧩
One page per prompt
Don't ask for the entire report in one shot. Build one page at a time, reload, verify, then move to the next. Iteration beats perfection.
💬
Ask Claude to explain
After generating JSON, ask "explain what each data role binding does here." This catches errors before you reload and builds your own PBIR knowledge.
🔁
Keep a backup before edits
Copy the pages/ folder before a large build session. If Claude writes malformed JSON that crashes Desktop, you can restore in seconds.
📏
Use a layout grid
Plan your canvas on paper first: 1280×720, 4 columns of ~300px. This makes writing coordinate values fast and prevents the "eyeball it" trap.
🚀
Imperfect output is fine
The goal isn't one-shot perfection — it's faster iteration. A rough first pass in 30 seconds beats an hour of drag-and-drop. Refine from there.
